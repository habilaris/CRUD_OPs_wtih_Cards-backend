Do you know that I am already building my own endpoints, already working with React+TailwindCSS+Express+CORS+NodeJS, I implemented three things of CRUD, mean Create, read, delete is done, just Update is pending, it is pseudo project, the frontend calls api on the express server that I run on different ports.

Even the UI is very polished, I put a lot of Effort in it, I even used React Router with Browser Router, Routes, Route, Navlink with isActive.

The Idea is simple, I have created a kind of a floating Navbar(That also unique and attractive), in that Navbar thing I made four options, create, read, update(update endpoint in express is pending), delete, these are options for a admin to create, read, update and delete user cards (I just needed a random idea to practice these CRUD ops).

For each option on UI I am calling a separate endpoint in backend server, in a separate JSX component, also giving each component (Create, Read, Update, Delete) a separate but elaborative route path for example "admin/create-card" etc with the help of react-router-dom.
